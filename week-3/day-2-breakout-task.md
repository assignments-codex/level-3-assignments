#### Goal

Practice user input with fetched data.

#### Steps

1. Complete what we built in lecture.
2. Add an input that lets the user search or filter the data.
3. Keep the original fetched data in state.
4. Render only the matching results on the page.
5. Keep the feature simple and easy to explain.

#### Extra practice if you want

- Add a reset button.
- Let the user search by more than one field.
- Show a message when no results match.
