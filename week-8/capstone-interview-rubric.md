# Level 3 Capstone - Interview Rubric

Use on Friday. Interview flow and scoring match your standard capstone format.

## Evaluator flow (10 to 12 minutes)

1. Live site demo: problem, solution, core flow
2. Code walkthrough: selected files and decisions
3. Q and A on code
4. Live coding: one small Level 3 task (no AI)
   Structure mirrors your existing evaluator guide so instructors can move fast.

## Scoring (100 points)

A. Live coding - 50 total

- Correct, runnable solution
- Basic edge case or input validation
- Clear, idiomatic JS (names, small functions)
- Think aloud while coding
- Quick self-check (console or test) and revise if needed

B. Explaining code - 30 total

- Design intent and trade offs (routes, components, state)
- Data flow and error handling (fetch, guards)
- Constraints and alternatives (rate limits, performance, keys)

C. Presentation and professionalism - 20 total

- Live link first; working demo
- Concise communication; on time
- Organized repo and README
- Clear Q and A

Same scale and categories you use in Level 2 and Level 4 capstones to keep expectations consistent.

## Live-coding prompts (pick one)

- Write or adjust a pure function for a given input and output
- Add or fix an effect that fetches data and handles loading and error
- Add a small route or controlled form and verify behavior
- Add a tiny test that asserts a utility or component output

Prompts mirror your capstone patterns and stay within Level 3 scope.
