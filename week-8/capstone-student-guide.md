# Level 3 Capstone - Student Guide

You will present a working project and complete a short live-coding task. Keep scope tight and show one reliable flow first.

## Objective

Ship a small React app and demonstrate a clean feature from start to finish.

## What you are building

- A React single page app deployed with a live URL
- One clear feature that uses routing, state, and an API call
- A short demo that shows the data flow in your app (UI to request to rendered result)

Use a simple theme. You are not graded on visuals. Focus on working code and clarity.

## Minimum requirements

- Live link and public repo
- At least two routes and one stateful component
- One API request with basic error handling (friendly message on failure)
- One small data transform or summary (map or filter or reduce)
- A minimal test that runs (utility or component), with a test script in package.json
- README with setup, live link, and a short architecture description

Optional

- A tiny component library or utility CSS
- One accessible form with labels and basic validation

Checklist adapted from your capstone format so students know exactly what to hit.

## Friday presentation (10 to 12 minutes)

1. Live site demo (about 2 min)
2. Code walkthrough (about 3 to 4 min)
3. Q and A on code (about 2 to 3 min)
4. Live coding (about 3 to 4 min, no AI) Follow the same flow you use elsewhere so students know what to expect.

## Live-coding demonstration

You will complete one short task picked by the instructor, for example:

- Write a pure function based on inputs and outputs
- Perform array processing with map or filter or reduce
- Add a small fetch call pattern with guards and render the result
- Render a safe DOM update or component output for a new state

Keep it short, speak your thinking, and do a quick console check if time allows.

## What to submit

- Live URL
- Repo URL
- README that includes:
  - One paragraph product brief
  - Setup and start steps
  - Short architecture overview (routes, state, where the API call lives)
  - Test script and how to run it

This mirrors your submission style from other capstones.

## Student rubric (100 points)

1. Live coding - 50

- Correct, runnable solution
- Handles a simple edge case or input validation
- Clear, idiomatic JS
- Think aloud while coding
- Quick self-check and adjust if needed

1. Explaining code - 30

- Design intent and trade offs
- Data flow and state or error handling
- Constraints and alternatives

1. Presentation and professionalism - 20

- Live link first and working demo
- Clear communication and timing
- Organized repo and README
- Confident Q and A

Same point split and expectations as your other capstones.
